CH 01

Understanding Local Networks

A Local Area Network (LAN) is a group of devices connected together in the same physical location — your home, building, or campus. When the internet is down, devices on the same LAN can still communicate with each other at full speed.

Key Concepts

TermMeaningExample
IP AddressUnique number for each device on the network192.168.1.5
SubnetGroup of IPs that can talk to each other directly192.168.1.0/24
Gateway/RouterDevice that connects local network to internet (or other networks)192.168.1.1
DHCPRouter automatically assigns IPs to devicesMost routers do this
MAC AddressHardware ID burned into every network adapterAA:BB:CC:DD:EE:FF
PortNumber that identifies a specific service on a device:8080
DNSTranslates names to IPs. Won't work offline unless you set it up!survival.local → 192.168.1.5

Find Your IP Address

ipconfig

Look for IPv4 Address under your network adapter (usually starts with 192.168.)

ip a
# or
hostname -I

Look for inet 192.168.x.x next to eth0 or wlan0

ifconfig | grep inet
# or
ipconfig getifaddr en0

Settings → WiFi → Tap your network → IP Address is shown in the details

Or install a network scanner app like "Network Analyzer"

Typical Home Network Layout

Router (192.168.1.1) — Internet | ─────┼────────────────────────────── | | | | Desktop Laptop Phone Tablet 192.168.1.2 .3 .8 .9 (Server) (Client) (Client) (Client)
In a survival scenario without internet, the router still works as a local switch — devices can still talk to each other even though there's no internet connection through it.

Web Server

A web server hosts files that any device can access via a browser. This is how you share this manual, HTML tools, and documents across your network.

Start the Web Server

START (serves from kit root on port 8080)
python scripts/serve_site.py
CUSTOM PORT AND DIRECTORY
python scripts/serve_site.py --port 9090 --dir /path/to/share

Connect From Other Devices

  1. Run the server on the host machine and note the IP shown (e.g. 192.168.1.5)
  2. Make sure the connecting device is on the same WiFi or LAN cable
  3. Open a browser on the other device
  4. Type: http://192.168.1.5:8080 (use your actual IP)
  5. You'll see the kit's file browser and this manual
Windows Firewall may block connections from other devices. If others can't connect, run: netsh advfirewall firewall add rule name="Survival HTTP" dir=in action=allow protocol=TCP localport=8080 — or temporarily disable the firewall.

Host Any Website Locally

Put any HTML file in the kit folder. For example, create mysite/index.html and it's accessible at http://192.168.1.5:8080/mysite/ from any device.

Open on Boot (Linux)

CREATE SYSTEMD SERVICE — Linux only
sudo nano /etc/systemd/system/survival-web.service

# Paste this:
[Unit]
Description=Offline Survival Kit Web Server
After=network.target

[Service]
ExecStart=/usr/bin/python3 /path/to/scripts/serve_site.py
Restart=always

[Install]
WantedBy=multi-user.target

# Then:
sudo systemctl enable survival-web
sudo systemctl start survival-web

CH 06

Custom Local Domains

Instead of remembering http://192.168.1.5:8080, you can use friendly names like http://survival.local:8080.

Method 1: Hosts File (Simplest)

The hosts file is a local DNS override. Edit it on every device that needs friendly names.

AUTOMATED — Run on each device that needs friendly names
# Windows: Run as Administrator
# Linux/Mac: Run with sudo
python scripts/setup_hosts.py
MANUAL — Edit hosts file directly
# Windows hosts file location:
C:\Windows\System32\drivers\etc\hosts

# Linux/Mac hosts file:
/etc/hosts

# Add these lines (replace 192.168.1.5 with your server IP):
192.168.1.5  survival.local
192.168.1.5  chat.local
192.168.1.5  ai.local
192.168.1.5  files.local

Method 2: mDNS / Avahi (Auto-Discovery)

mDNS lets devices find each other by hostname.local automatically — no configuration needed per device.

# Install Avahi (mDNS daemon)
sudo apt install avahi-daemon

# Set your hostname
sudo hostnamectl set-hostname survival-server

# Now other Linux/Mac devices can reach you as:
# http://survival-server.local:8080

Windows 10/11 includes mDNS support built-in. Your computer can be reached as COMPUTERNAME.local. Check your computer name in System Properties.

Android supports .local addresses natively on most versions. Try typing survival-server.local in your browser. If it doesn't work, use the IP address instead or use a DNS Changer app.

Method 3: Run a DNS Server

For larger networks, run a proper DNS server so all devices get domain names automatically.

DNSMASQ — lightweight DNS for your network
# Install
sudo apt install dnsmasq

# Edit config: /etc/dnsmasq.conf
address=/survival.local/192.168.1.5
address=/chat.local/192.168.1.5
address=/ai.local/192.168.1.5

# Restart
sudo systemctl restart dnsmasq

# Tell your router to use this machine as its DNS server
# Or configure each device to use 192.168.1.5 as DNS
CH 07

Creating a Local Network

Option A: Use an Existing Router (Easiest)

If you have a WiFi router, even without internet connection it still creates a LAN. Just connect devices to its WiFi or LAN ports. The router acts as a DHCP server assigning IPs automatically.

  1. Plug the router into power
  2. Connect devices to its WiFi (network name / SSID is on the router sticker)
  3. All connected devices can communicate with each other
  4. Find your IP with ipconfig (Windows) or ip a (Linux)
  5. Run the server scripts from this kit

Option B: Direct Cable Connection (2 Devices)

Connect two computers directly with an Ethernet cable (use a crossover cable or modern adapters auto-detect).

Set static IPs for direct connection — Linux
# On Machine 1:
sudo ip addr add 10.0.0.1/24 dev eth0
sudo ip link set eth0 up

# On Machine 2:
sudo ip addr add 10.0.0.2/24 dev eth0
sudo ip link set eth0 up

# Test:
ping 10.0.0.1

Option C: WiFi Hotspot (Phone as Router)

Any smartphone can create a WiFi hotspot that other devices connect to — no router needed.

Settings → Network → Hotspot & Tethering → WiFi Hotspot → Turn on

Set a password, then other devices connect to your phone's hotspot. Your phone will be the gateway (usually at 192.168.43.1).

# Windows 10/11 — Mobile Hotspot
Settings → Network & Internet → Mobile Hotspot → On

# Or via command line:
netsh wlan set hostednetwork mode=allow ssid="SurvivalNet" key="password123"
netsh wlan start hostednetwork
# Using NetworkManager
nmcli device wifi hotspot ssid "SurvivalNet" password "password123"

# Or with hostapd (more control)
sudo apt install hostapd dnsmasq
# Configure /etc/hostapd/hostapd.conf

Assigning Static IPs (Important for Servers)

Server machines should have a fixed (static) IP so client devices always know where to connect. Otherwise the DHCP server may give a different IP each time.

1. Settings → Network → Adapter Options
2. Right-click your adapter → Properties
3. IPv4 → Properties
4. Set:
   IP Address  : 192.168.1.5
   Subnet Mask : 255.255.255.0
   Gateway     : 192.168.1.1 (your router)
   DNS         : 192.168.1.1 (or leave blank)
# Permanent — edit /etc/netplan/01-netcfg.yaml (Ubuntu/Debian)
network:
  version: 2
  ethernets:
    eth0:
      addresses: [192.168.1.5/24]
      gateway4: 192.168.1.1
      nameservers:
        addresses: [192.168.1.1]

sudo netplan apply

# Or via NetworkManager:
nmcli con mod "Wired" ipv4.addresses 192.168.1.5/24
nmcli con mod "Wired" ipv4.gateway 192.168.1.1
nmcli con mod "Wired" ipv4.method manual
nmcli con up "Wired"
CH 08

Mesh Networking

What is a Mesh Network?

In a standard network, all devices connect to ONE central router. If the router fails, the whole network dies. In a mesh network, every device is also a router — there is no single point of failure.

STANDARD NETWORK: MESH NETWORK: [Router] [Node A]──[Node B] / | \ \ X / A B C [Node C]──[Node D] (if router dies → all die) (any path can route around broken nodes)

Why Use a Mesh Network?

  • Range extension: Each node extends WiFi range — a chain can span kilometers
  • No single point of failure: If one node dies, traffic reroutes automatically
  • Scalability: Add more nodes to expand coverage
  • Resilience: Critical for disaster/war scenarios where infrastructure is destroyed
  • Self-healing: Network finds the best path automatically

The Chain Effect — Long Range

Devices in a mesh relay messages through each other. A signal can travel:

House ──[300m]── Relay Node ──[300m]── Another House ──[300m]── ... (WiFi range) (battery-powered) Each hop extends range by ~300m (outdoor) 10 nodes = ~3km coverage chain

Software Mesh: batman-adv (Linux)

B.A.T.M.A.N. Advanced (Better Approach To Mobile Adhoc Networking) is the best open-source mesh protocol, built into the Linux kernel.

SETUP BATMAN-ADV MESH — Linux (each node)
# Install
sudo apt install batctl

# Load kernel module
sudo modprobe batman-adv

# Configure wireless adapter for ad-hoc mode
sudo ip link set wlan0 down
sudo iw dev wlan0 set type ibss
sudo ip link set wlan0 up
sudo iw dev wlan0 ibss join "SurvivalMesh" 2412

# Add wlan0 to batman-adv
sudo batctl if add wlan0

# Bring up the batman interface
sudo ip link set bat0 up
sudo ip addr add 10.0.0.X/8 dev bat0   # unique IP per node

# Check mesh status
sudo batctl n     # show neighbors
sudo batctl o     # show routing table

Easy Mesh with OpenWRT Routers

If you have spare WiFi routers, flash them with OpenWRT firmware and enable BATMAN-ADV or 802.11s mesh mode. This is the most robust approach.

OPENWRT MESH SETUP (on each router)
# SSH into the router (default: ssh root@192.168.1.1)
opkg update
opkg install kmod-batman-adv batctl

# Configure /etc/config/network
config interface 'mesh'
    option proto 'batadv'
    option mesh 'bat0'

config interface 'bat0_dev'
    option proto 'batadv_hardif'
    option master 'mesh'
    option mtu '1536'

Meshtastic — Long-Range Radio Mesh

For truly long-range mesh (without WiFi), Meshtastic uses cheap LoRa radio modules (€10–€30 each) that can reach 5–15km per hop in open terrain.

Quick Mesh Design Rules

  • Nodes should be line-of-sight for maximum range
  • Elevate nodes (rooftops, hills) for much better range
  • Use directional antennas between fixed nodes, omnidirectional for user access
  • Assign static IP addresses to each node (no DHCP server needed in mesh)
  • Plan for battery or solar power at each node for resilience
  • Always have at least 2 paths between important nodes
CH 09

WiFi Router Integration

Adding routers to your network extends wireless coverage to more devices without cables.

Mode 1: Router as Access Point (Recommended)

Connect a second router via Ethernet to your main router/switch. Configure it as an access point (AP) — it extends WiFi without creating a separate network.

  1. Connect Ethernet cable: LAN port of main router → LAN port of second router (not WAN/Internet port)
  2. Log into second router admin panel (usually at 192.168.1.1 or 192.168.0.1)
  3. Disable DHCP server on the second router (very important — only one DHCP server per network)
  4. Set a static IP for the second router (e.g. 192.168.1.2)
  5. Set the WiFi SSID and password (can be same as main router for seamless roaming)
  6. Devices connecting to either router are on the same network

Mode 2: Router as Repeater/Extender

Some routers support WDS (Wireless Distribution System) or repeater mode to extend WiFi wirelessly — no cable needed.

Wireless repeaters halve bandwidth at each hop. Use wired access points when possible for better performance.

Mode 3: Multiple Separate Networks (Routing)

For larger setups (different buildings, floors), each router creates its own subnet. You'll need to configure routing between them.

Building A Building B Building C 192.168.1.0/24 192.168.2.0/24 192.168.3.0/24 [Router A] ──── [Central Switch] ──── [Router B] | [Router C]

Switch vs Router vs Access Point

DeviceWhat It DoesWhen to Use
SwitchConnects multiple devices with cables, one networkWired expansion, more ports
RouterConnects networks together, has DHCPCreating a new network segment
Access PointWiFi radio connected to existing networkExtending wireless coverage

Channel Planning (Reduce Interference)

2.4 GHz non-overlapping channels5 GHz
Channels 1, 6, 11Many non-overlapping channels (36, 40, 44, 48…)
Set neighboring access points to different non-overlapping channels. 5 GHz has shorter range but much less interference and higher speed — prefer it for high-density areas.

Download Python Packages Offline

Before the internet goes down, download all needed packages as .whl files so you can install them offline later:

DOWNLOAD PACKAGES WHILE ONLINE (on any machine)
# Download all wheels to a folder
pip download pyftpdlib websockets flask flask-cors requests -d ./offline-packages

# Later, install from that folder (offline):
pip install --no-index --find-links=./offline-packages pyftpdlib websockets flask flask-cors requests
